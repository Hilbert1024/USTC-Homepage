学习笔记

基本单位

位与字节

  • 1字节 = 8位，1字 = 2字节 = 16位
  • 字节用byte、B表示，位用bit、b表示，字用word表示
  • 用 b、w、l、q 分别表示1字、2字、4字、8字
  • 指针在32位机器上大小为4 byte，在64位机器上为8 byte

x位结构

x位结构描述了一个CPU具有以下几方面的特征结构：

  • 寄存器最大宽度为x位
  • 运算器一次最多可处理x位的数据
  • 寄存器和运算器之间的通路为x位

寄存器与内存

  • 内存中一个地址指向的空间为1字节，即存储器中的一个储存单元为1字节
  • 根据不同功能将内存分为数据段、代码段、堆栈段、拓展段等
  • 常用寄存器名称及用途

  • 当参数个数超过6个时，用内存的栈帧结构保存变量

  • push、pop指令

    push 寄存器 ：将一个寄存器中的数据入栈

    pop 寄存器 ：出栈，用一个寄存器接收出栈的数据

    push 内存单元 ：将一个内存单元处的字入栈

    pop 寄存器 ：出栈，用一个内存字单元接收出栈的数据

    下面两段指令等价：

    pushq %rbp
    xxxxxxxxxx
    subq 8, %rsp
    movq %rbp, (%rsp)

  • 被调用者保存寄存器：

    P 调用 Q 时，被调用者 Q 需要保证被调用者保存寄存器的值不变，有以下两种方法

    1. 根本不去改变寄存器的值
    2. 将原始值压入栈中，在返回前从栈中弹出旧值

  • 对于储存在RAM某存储单元的变量x，x表示该存储单元的值，&x为该存储单元的地址

    *x有如下用途：

    1. 声明指针（Pointer Statement）： int x =5; int* ptr=&x ; 就是声明变量x是5，把x的地址附到指针ptr上
    2. 解引用 （Dereference）： *ptr 单独拿出来就是找出 ptr 指针指向的值
    xxxxxxxxxx
    int x = 10;
    int *px = &x;
    printf(“%d\n”, x);   //10
    printf(“%d\n”, &x);  //6487620
    printf(“%d\n”, px);  //6487620
    printf(“%d\n”, *px); //10

  • 一般地，地址存放于寄存器中，我们用一个例子说明。若

    xxxxxxxxxx
    #include <stdio.h>
    int main(void)
    {
        int a = 0x7F00;
        int *p = &a;
        return 0;
    }

    执行gcc -S test.c，其汇编语言为

    xxxxxxxxxx
    main:
        pushq   %rbp
        movq    %rsp, %rbp
        subq    $48, %rsp
        call    __main
        movl    $32512, -12(%rbp)
        leaq    -12(%rbp), %rax
        movq    %rax, -8(%rbp)
        movl    $0, %eax
        addq    $48, %rsp
        popq    %rbp
        ret

    可以看到

    int a = 0x7F00 <=> movl $32512, -12(%rbp)

    int *p = &a <=> leaq -12(%rbp), %rax

    此时寄存器%rax中保存的即为a的值。

指针

  • 每个指针都对应一个类型。

    xxxxxxxxxx
    int *ptr1; // 指向类型为int的指针
    char **ptr2; // 指向一个指向类型为char的指针的指针

  • 每个指针都有一个值。

    这个值是某个指定类型对象的地址。

  • 指针用 '&' 运算符创建。

    '&' 运算符的机器码实现常常用leaq这条指令来计算表达式的值。

    对于声明的数组

    T D[R][C],

    其数组元素 D[i][j] 的内存地址为

    &D[i][j]=xs+L∗(C∗i+j).

  • * 操作符用于间接引用指针。

    xxxxxxxxxx
    Expr == *&Expr; // true

  • 数组与指针紧密联系。

    xxxxxxxxxx
    A[i] == *(A + i); // true

  • 将指针从一种类型强制转换成另一种类型，只改变它的类型，而不改变它的值。

    强制类型转换的一个效果是改变指针运算的伸缩。

    xxxxxxxxxx
    char *p;
    (int *)p + 7 == p + 28; // true
    (int *)(p + 7) == p + 7; // true

  • 指针也可以指向函数。

    xxxxxxxxxx
    int fun(int x, int *p);
    int (*fp)(int, int *);
    fp = fun; // 将函数赋给指针
    int y = 1;
    int result = fp(3, &y); // 可以这样调用函数

    函数指针的值是该函数机器代码表示中第一条指令的地址。

异质的数据结构

  • struct：将多个对象集合到一个单位中
  • union：用几种不同的数据类型来引用一个对象

结构

  • 示例：

    xxxxxxxxxx
    struct hero {
        int hp;
        int mp;
    }
    // 声明
    struct hero h;
    // 赋值
    h.hp = 1000;
    h.mp = 1000;
    // 还可以用以下表达式初始化
    struct hero h = {1000, 1000};
    // 将结构体指针传入函数

        return (*h).hp * 2;
    }
    // 引用结构体的元素可简化为
    int double_hp(struct hero *h) {
        return h->hp * 2;
    }

联合

联合提供了一种方式，能够规避C语言的类型系统，允许以多种类型来引用一个对象。

  • 若事先知道对一个数据结构中的两个不同字段的使用是互斥的，那么将这两个字段声明为联合的一部分，而不是结构的一部分，会减小分配空间的总量。

结构与联合的差别

  • 结构体结构体的各个成员会占用不同的内存块，互相之间没有影响；而联合体的所有成员占用同一段内存块，修改其中一个成员会对整个内存块保存的值产生影响。

运行时栈

转移控制

  • call的效果是将call的下一条指令地址A压入栈中，随后跳转至call所指定的函数地址。遇到ret指令返回时，从栈中弹出指令A并跳转到该地址。
  • %rip为程序计数器，记录目前程序执行的地址

不同语言的编译

  • 编译 C++ 与编译 C 非常相似。C++ 的对象用结构来表示，类似于 C 的 struct。
  • C++ 的方法是用指向实现方法的代码的指针来表示的。
  • Java 的实现方式完全不同，其目标代码是一种特殊的二进制表示，称为 Java 字节代码。这种代码可以看成是虚拟机的机器级程序。
  • 还有一种称为及时编译（JIT）的方法，动态地将字节代码序列翻译成机器指令。当代码要执行多次时，这种方法执行起来更快。

优化程序性能

错误通常源自程序员没想到的情况。

如何做

  1. 选择一组适当的算法和数据结构
  2. 编写出编译器能够有效优化以转换成高效可执行代码的源代码
  3. 针对处理运算量特别大的计算，将一个任务分成多个部分，这些部分可以在多核和多处理器的某种组合上并行地计算。

妨碍优化

  1. 内存别名使用（memory aliasing）：需考虑两指针指向同一位置的情况
  2. 函数调用：内联函数

表示程序性能

  • CPE （每元素的周期数）：度量计算每个元素在循环过程中消耗的时钟周期，表现迭代程序的循环性能。值越小表面性能越好。
  • 使用最小二乘法拟合一系 (元素个数, 消耗周期) 的点，得到的系数即是CPE。

优化方法

  1. 消除循环的低效率：可将循环体内不变的值移动至循环体外。
  2. 减少过程调用：可将循环体内的过程调用移动至循环体外。
  3. 消除不必要的内存引用：在循环体中执行指针运算，会频繁调用从内存读出再写入内存。查看下面的例子：
xxxxxxxxxx
void slow(double* data[], double *dest) {
    *dest = 1;
    for (int i = 0; i < 100; i++) {
        *dest = *dest * data[i];
    }
}

优化后：

xxxxxxxxxx
void fast(double* data[], double *dest) {
    double acc = 1;
    for (int i = 0; i < 100; i++) {
        acc *= data[i];
    }
    *dest = acc;
}

在运行时，函数 slow 会使用一个额外的寄存器储存*dest的值。它将 dest写入内存，执行乘法操作后再从内存独出该值，写入寄存器。这样一来，每次计算都会进行内存读写操作，极大浪费了性能。

功能单元的性能

每个运算都由以下数值来刻画：

  • 延迟：完成运算所需要的总时间——执行实际运算所需要的时钟周期总数。
  • 发射时间：两个连续的同类型的运算之间需要的最小时钟周期数。
  • 容量：能够执行该运算的功能单元的数量——同时能发射多少个这样的操作。

容量为C，发射时间为I，则吞吐量为每时钟周期 C/I 个操作。

处理器操作的抽象模型

  • 数据流图：识别出循环寄存器，形成数据流图。
  • 较少的操作并不一定能带来较快的性能，参考 5.5 和 5.6。
  • 数据流表示中的关键路径提供的只是程序需要周期数的下界。
  • 关键路径指明了执行该程序所需时间的下界。如果程序中有某条数据相关链，这条链上的所有延迟之和等于 T，那么这个程序至少需要 T 个周期才能执行完。
  • 数据相关指的是寄存器值的变化。

As a tool for analyzing the performance of a machine-level program executing on a modern processor, we will use a data-flow representation of programs, a graphical notation showing how the data dependencies between the different operations constrain the order in which they are executed. These constraints then lead to critical paths in the graph, putting a lower bound on the number of clock cycles required to execute a set of machine instructions.

循环展开

  • 在循环体内将操作拆分为 k 次，称为 k * 1 循环展开。
  • k = 2，迭代次数减半了，但每次迭代还是有两个顺序的操作。
  • 编译器可以很容易地执行循环展开。

提高并行性

  • 对于迭代累乘，可以分解为两路乘法，使得每条关键路径的操作减半，CPE 降低 50%。此处 k = 2。
  • 可以将多个累积变量变换归纳为将循环展开 k 次，以及并行累积 k 个值，得到 k*k 循环展开。这样优化的程序可以得到理论界限，为吞吐量界限。
  • 若要达到吞吐量界限，对延迟为L，容量为C的操作而言，要求循环展开因子 k >= C * L 。

限制因素

  • 寄存器溢出：当并行度 p 超过了可用的寄存器数量，那么编译器会诉诸溢出。有时增加并行量不能带来性能优化，甚至还会导致性能变差。

  • 分支预测和预测错误处罚：在写程序时，可遵循以下原则：

    1. 不要过分关心可预测的分支
    2. 书写适合用条件传送实现的代码，例如用条件数据传送代替条件控制

内存性能

  • 加载：从内存读到寄存器
  • 存储：从寄存器写到内存。不改变寄存器的值，就其本性来说，一系列存储操作不会产生数据相关。
  • 写/读相关：一个内存读的结果依赖于一个最近的内存写，这会使得操作不能并行，显著降低性能。

加载和存储执行单元：

  • 存储缓冲区（Store buffer）：包含已经被发射到存储单元而又还没有完成的存储操作的地址和数据，这里的完成包括更新数据高速缓存。
  • 加载操作发生时，必须先检查储存缓冲区中的条目，看有没有地址相匹配。如果有（意味着在写的字节与在读的字节有相同的地址），它就取出相应的数据条目作为加载操作的结果。

观察以下代码：

xxxxxxxxxx
movq %rax, (%rsi)
movq (%rdi), %rax

movq %rax, (%rsi) 被翻译成两个操作：

  1. s_data : 在存储缓冲区创建一个条目，并设置该条目的地址字段
  2. s_addr : 设置该条目的数据字段

两操作是独立执行的，执行后存储缓冲区部分被修改为：

movq (%rdi), %rax 译码得到的 load 操作必须检查所有未完成的存储操作序列。当检查存储缓冲区时，若

  1. %rsi 与 %rdi 相同：load 操作必须等待直到 s_data 将它的结果存放到存储缓冲区中。
  2. %rsi 与 %rdi 不同：两个操作就可以独立地执行。

对于两地址相同的情况，即写读相关，会显著地降低性能，因为有了数据相关，关键路径变长。

总结：提高性能技术

  1. 高级设计：选用适当的算法和数据结构。

  2. 基本编码原则：

    • 消除连续的函数调用：将计算移动到循环外。
    • 消除不必要的内存引用：引用临时变量来保存中间结果，仅在最后将结果放至数组或全局变量中。

  3. 低级优化：

    • 循环展开。
    • 通过多个累积变量和重新结合等技术，找到方法提高指令级并行。
    • 用功能性的风格重写条件操作，使得编译器采用条件数据传送。

存储器层次结构

随机访问存储器

  • RAM：Random-Access Memory，随机访问存储器
  • SRAM：Static. 将每个位存储在一个双稳态的存储单元里。稳定。
  • DRAM：Dynamic. 将每个位存储为对一个电容的充电。不稳定。
  • DRAM芯片构造：由 d 个超单元组成，每个超单元有w位。超单元被组织成一个 r 行 c 列的长方形阵列，rc = d。
  • DRAM芯片封装在内存模块中，插在主板的扩展插槽上。
  • 通过将多个内存模块连接到内存控制器，能够聚合成主存。在这种情况中，当控制器收到一个地址A时，控制器选择包含A的模块k，将A转换成它的 (i, j) 的形式，并将 (i, j) 发送到模块k。

 

  • 增强型DRAM：

    • 双倍数据速率同步DRAM (Double Data-Rate Synchronous DRAM, DDR SDRAM)：通过使用2个时钟沿作为控制信号，从而使DRAM的速度翻倍。
    • 视频RAM (Video RAM, VRAM)

  • 闪存 (flash memory)：非易失性存储器，即断电后仍可保存信息。其中有一种基于闪存的磁盘驱动器被称为固态硬盘 (SSD)。

  • 存储在ROM设备中的程序通常被称为固件 (firmware)。当一个计算机系统通电后，它会运行存储在ROM中的固件。

  • 总线：数据流通过称为总线的共享电子电路在处理器和DRAM主存之间来来回回。下图是总线设计的一个抽象，具体细节有差异。当我们将一个地址A加载至寄存器中时会发生以下步骤：

    xxxxxxxxxx
    movq A, %rax
    1. CPU芯片上称为总线接口 (bus interface) 的电路在总线上发起读事务。读事务包含接下来三个步骤。
    2. CPU将地址A放到系统总线 (system bus) 上。I/O 桥 (I/O bridge) 将信号传递到内存总线 (memory bus)。
    3. 主存感知到内存总线上的地址信号，从内存总线读地址，从DRAM取出数据字，并将数据写到内存总线。I/O 桥将内存总线信号翻译成系统总线信号，然后沿着系统总线传递。
    4. CPU感知到系统总线上的数据，从总线上读数据，并将数据复制到寄存器 %rax。

 

磁盘

逻辑磁盘

  • 逻辑磁盘封装了物理磁盘的特性，将磁盘的构造呈现为一个简单的视图，一个 B 个扇区大小的逻辑块的序列，编号为 0, 1, 2, ... , B -1.
  • 磁盘封装中有一个小的硬件/固体设备，称为磁盘控制器，维护着逻辑块号和实际磁盘扇区之间的映射关系。
  • 当操作系统想要执行I/O操作时，会发送一个命令到磁盘控制器，磁盘控制器解析出三元组 (盘面，磁道，扇区) 来唯一确定数据在磁盘上的位置。磁盘控制器会解释这个三元组，将读写头移动到适当的柱面，等待扇区移动到读写头下，将读写头感知到的位放到控制器上的一个小缓冲区中，然后将它们复制到主存中。

I/O总线

 

  • 图形卡、监视器、鼠标、键盘等这样的 I/O 设备，都是通过 I/O 总线连接到 CPU 和主存的。
  • CPU使用一种称为内存映射 I/O 的技术来向 I/O 设备发射命令。在使用内存映射 I/O 的系统中，地址空间中有一块地址是为与 I/O 设备通信保留的。每个这样的地址称为一个 I/O 端口。

局部性

  • 一个编写良好的程序常常具有良好的局部性。
  • 时间局部性：被引用过一次的内存位置在不久被多次引用。
  • 空间局部性：一个内存位置被引用了一次，在不久其附近的内存位置被引用。
  • 顺序访问一个向量每个元素的函数，具有步长为1的引用模式，称为顺序引用模式。步长为 k 的访问就称为步长为 k 的引用模式。
  • 如何看一眼源代码就能获得对程序中局部性的高层次的认识，是程序员要掌握的一项有用而且重要的技能。

缓存

  • 层次结构中的每一层都缓存来自较低一层的数据对象。
  • 第 k 层的缓存包含 k+1 层块的一个子集的副本。
  • 数据总是以块大小为传送单元在第 k 层和第 k+1 层之间来回复制的。

缓存命中：

当程序需要第 k+1 层的某个数据对象 d 时，它首先在当前存储在第 k 层的一个块中查找 d。如果 d 刚好缓存在第 k 层中，那么就是缓存命中。

  • 当发生缓存不命中时，第 k 层的缓存从第 k+1 层缓存中取出包含 d 的那个块，如果第 k 层的缓存已经满了，可能就会覆盖现存的一个块。
  • 大多数时候，缓存都是自动运行的，不需要程序采取特殊的或显示的行动。

高速缓存

读

  • 高速缓存由 (S, E, B, m) 描述。

     

  • 高速缓存地址将m个位分为t(标记)，s(组索引)，b(块偏移)。m=t+s+b.

  • 当CPU发出一个地址A时，可通过s找到数据所在的组，通过t找到数据所在的行，通过b找到数据在高速缓存块中的偏移。由

S=2s, B=2b, t=m−(s+b)

可以知道该查找是充分的。

  • 高速缓存的大小为

    C=S×E×B.

  • E = 1 时被称为直接映射高速缓存。

  • 高速缓存确定一个请求是否命中，然后抽取出被请求的字的过程，分为三步：1. 组选择 2. 行匹配 3. 字抽取。

    Step1：直接映射高速缓存中的组选择。从地址中读取组索引，并定位到所在的组。

    Step2：直接映射高速缓存中的行匹配。上一步选择了某个组i，现在确定是否有字w的一个副本存储在组i包含的一个高速缓存行当中。如果查找到某行的标记位与地址中的标记位匹配，则缓存命中；否则不命中。

    Step3：直接映射高速缓存中的字选择。命中后我们就知道字w就在这个块中的某个地方。通过偏移来定位字的位置。

    Step4：直接映射高速缓存中不命中时的行替换。如果缓存不命中，那么它需要从存储器层次结构中的下一层取出被请求的块，然后将新的块存储在组索引位指示的组中的一个高速缓存行中。如果组中都有高速缓存行了，则要驱逐一块。

  • 组相联高速缓存满足

    1<E<C/B.

  • 组相联高速缓存中每组有多行。相联存储器是一个键值对，可以把每组看成一个小的相联存储器，键是标记和有效位，值就是块的内容。

  • 组相联高速缓存中不命中时，需要替换存在的行。若有空行则替换之；否则将使用一定的策略来替换行。策略有LFU、LRU等。

  • 全相联高速缓存满足

    E=C/B, i.e. S=1.

  • 全相联高速缓存只有一个组，故其地址不包含组索引，由t位标记和b位块偏移构成。

参数性能影响

虚拟内存

  • 虚拟内存（virtual memory，VM）又叫做虚拟存储（virtual storage)，是一种内存管理技术。它是操作系统提供的一种对主存的抽象。
  • 为了更加有效地管理内存并减少出错，现代操作系统提供了一种对主存的抽象概念，即是虚拟内存（Virtual Memory）。虚拟内存为每个进程提供了一个一致的、私有的地址空间，它让每个进程产生了一种自己在独享主存的错觉（每个进程拥有一片连续完整的内存空间）。

虚拟内存主要提供了如下三个重要的能力：

  • 它把主存看作为一个存储在硬盘上的虚拟地址空间的高速缓存，并且只在主存中缓存活动区域（按需缓存）。-
  • 它为每个进程提供了一个一致的地址空间，从而降低了程序员对内存管理的复杂性。
  • 它还保护了每个进程的地址空间不会被其他进程破坏

物理和虚拟寻址

  • 计算机系统的主存被组织成一个由M个连续的字节大小的单元组成的数组，每字节都有一个唯一的物理地址。物理地址长度即为前文的m。

  • CPU访问内存的最自然的方式就是使用物理寻址。我们把这种方式称为物理寻址。

  • 使用虚拟寻址，CPU通过生成一个虚拟地址来访问主存，这个虚拟地址在被送到内存之前先转换成适当的物理地址。

  • CPU芯片上叫做内存管理单元（MMU）的专用硬件，利用存放在主存中的查询表来动态翻译虚拟地址，该表的内容由操作系统管理。

  • CPU从一个有N=2n个地址的地址空间中生成的虚拟地址，这个地址空间称为虚拟地址空间

    {0,1,2,...,N−1}.

  • 物理地址空间对应于系统中物理内存的M个字节，M不要求是2的幂，但为简化讨论一般假设M=2m。

  • 地址空间区分了数据对象（字节）和它们的属性（地址）。

虚拟内存作为缓存的工具

  • 虚拟内存被组织为一个由存放在磁盘上的N个连续的字节大小的单元组成的数组。每字节都有一个唯一的虚拟地址。

  • 虚拟页被分为三个不相交的子集：

    • 未分配的
    • 已缓存的
    • 未缓存的

虚拟内存作为内存管理工具

  • 操作系统为每个进程提供了一个独立的页表，因而也就是一个独立的虚拟地址空间。
  • 多个虚拟页面可映射到同一个物理页面上。
  • malloc时操作系统分配一个k个连续的虚拟内存页面，映射到物理内存中k个页面，而物理内存中的页面无需是连续的。页面可以随机地分散在物理内存中。

虚拟内存作为内存保护的工具

地址翻译

  • 地址翻译可表示为一个映射
MAP:VAS→PAS∪ϕ,

这里VAS为虚拟地址空间中的元素，PAS为物理地址空间中的元素。

  • n位虚拟地址可表示为

    VA=(vn−1,...,vp+1,vp,⏟Virtual Page Number vp−1,...,v1,v0)⏟Virtual Page Offset.

  • MMU利用VPN来选择适当的PTE。

  • 由于VA的前n-p位为VPN，且2n−p为PTE个数，故可用VPN定位PTE位置。例如VPN选择x则PTE选择x。

  • 页大小为P=2p，故可用VPO(p位)定位页面偏移量。

  • 将页表中的物理页号（PPN）和虚拟地址中的VPO串联起来，就得到相应的物理地址。PPO=VPO。

虚拟地址 OR 物理地址？

  • 大多数系统是采用物理寻址的。
  • 利用TLP加速地址翻译。TLP可理解为在CPU上的PTE缓存。

多级页表

  • PTE个数 = 虚拟地址空间大小 / 页大小

  • 页表大小= PTE个数 * PTE大小

  • 32位地址空间，4KB页面，一个4字节的PTE，页表需要占据内存空间为

    232/(4∗210)∗4B=220∗4B=4MB.

  • 可通过划分VPN来定位多级页表，这是由VPN位数与PTE个数的关系得出的

    #PTE=2n−p.

    将 n−p 拆分为 k 组，每组长度为 l1,l2,...,lk，我们有

    ∑i=1kli=n−p.

    则每级PTE的个数为

    #PTEi=2li.

    若给我们一个虚拟地址 (vn−1,...,vp,vp−1,...,v0)，首先可通过前 l1 位定位 一级页表PTE的地址（一级页表的表头地址是由页表基址寄存器给出的）；一级页表PTE给出了二级页表表头地址，再通过 l1 后面紧跟的 l2 位偏移量确定2级PTE表的地址。重复k次可最终得出k级PTE表的位置，而k级PTE表保存了物理页号PPN，这样我们就通过虚拟地址找到了物理地址。物理地址的后p位由虚拟地址的后p位给出。

  • 上述方法的优点：

    1. 如果一级页表中的一个PTE是空的，那么相应的二级页表就不存在，这样潜在节省了大量内存。
    2. 只有一级页表才需要总是在主存中；虚拟内存系统可以在需要时创建、页面调入或调出二级页表。

页连续

  • 程序可以使用一系列连续的虚拟地址来访问物理内存中不连续的大内存缓冲区。
  • 程序可以使用一系列虚拟地址来访问大于可用物理内存的内存缓冲区。 当物理内存的供应量变小时，内存管理器会将物理内存页（通常大小为 4 KB）保存到磁盘文件。 数据或代码页会根据需要在物理内存与磁盘之间移动。
  • 不同进程使用的虚拟地址彼此隔离。 一个进程中的代码无法更改正在由另一进程或操作系统使用的物理内存。
  • 由地址翻译过程可知，将虚拟地址翻译为物理地址时，随着虚拟地址的增加，物理地址的后 p 位是连续增加的，而这 p 位组成了大小为 P=2p 的物理连续空间，这正是定义的一页的大小。所以我们可以将主存以页为单位进行划分，页可以整个保存在磁盘中。
  • 对于一个页大小为4KB（VPO为12位），L1 Cache有64个组和大小为64KB的缓存块（即物理地址的后12位为组索引和偏移位）的计算机，VPO位数恰好与组索引与缓存偏移位的和相同，这不是偶然。这样设计的好处是在MMU进行地址翻译时，通过VPO查找L1 Cache数据可同步进行，即加速了对L1 Cache的访问。

Linux虚拟内存系统

  • Linux为每个进程维护了一个单独的虚拟地址空间。
  • 每个存在的虚拟页都保存在某个区域中，而不属于某个区域的虚拟页是不存在的。

内存映射

  • 一个区域可以映射到一个匿名文件，匿名文件是由内核创建的，包含的全是二进制零。当CPU第一次引用这样一个区域内的虚拟页面时，内核就在物理存储器中找到一个合适的牺牲页面，如果该页面被修改过，就将这个页面换出来，用二进制0覆盖牺牲页面，并更新页表，将这个页面标记为是驻留在存储器中的。因为这个原因，映射到匿名文件的区域中的页面，有时也叫做请求二进制0的页。

在内核里,用户空间的进程要访问内存或磁盘里的数据要通过映射的方式将内存的物理地址和用户空间的虚拟地址联系起来. 用户通过访问这样的虚拟地址就可以访问到实际的物理地址,也就是实际的物理内存. 映射在实现虚拟地址到物理地址中扮演重要角色. 内核中映射分为文件映射和匿名映射. 文件映射就是磁盘中的数据通过文件系统映射到内存再通过文件映射映射到虚拟空间.这样,用户就可以在用户空间通过 open ,read, write 等函数区操作文件内容. 匿名映射就是用户空间需要分配一定的物理内存来存储数据,这部分内存不属于任何文件,内核就使用匿名映射将内存中的某段物理地址与用户空间一一映射,这样用户就可用直接操作虚拟地址来范围这段物理内存. 至于实际的代码,文件映射的操作就是: open,read,write,close,mmap... 操作的虚拟地址都属于文件映射. malloc 分配的虚拟地址属于匿名映射.

1.什么是CPU第一次引用这样一个区域内的虚拟页面，比如那个区域？

CPU 第一次访问匿名映射的空间,所谓的访问就是读或者写这部分区域. 根据 glibc 也就是用户空间对 malloc 分配的内存,如果不进行实际的访问,内核只分配虚拟地址,而不将虚拟地址通过匿名映射与物理地址一一对应.只有 CPU 实际访问了这些区域,才真正发生匿名映射,也才发生缺页处理. CPU 第一次引用匿名区域时,由于匿名虚拟区还没有和实际的物理区域映射,内核就会发生一个缺页错误.于是内核就会 通过一些策略,从 swap 区获得一块物理内存页与之映射.这样虚拟地址就和物理地址挂钩,也就会生成相应的页表, 页表相关信息会被存储到 TLB 或 CACHE 中以假设虚拟地址到物理地址的转换.

2.驻留在存储器中是标记为已分配未缓存吗？

内核为了管理内存,会将物理内存分成 4K 大小的数据块进行管理,如果这块物理页已经分配出去了,这样的统计计数就会加 1, 其状态就会变成使用中,其也会被加入到正在使用物理页链表中.由于该页已经通过匿名映射分配出去了.

3.映射到匿名文件区域页面什么意思？可以举个例子吗？

对于页面这个概念,虚拟空间,内核有 0 ~ 4G 的虚拟地址,这些虚拟地址被分作大小 4K 的区域.这些区域要做页面. 对于实际物理内存,其也被分成大小为 4K 的区域,这些区域叫做物理页帧,用结构体 struct page 进行管理. 问题中所说的匿名文件页面指的就是物理内存的物理页帧,代码中表示为 struct page.

4.我们常说的swap区域是匿名文件吧，由内核创建的吗？ 为什么swap区域的大小限制着当前运行着的进程能够分配的虚拟页面总数？

swap 区叫做交换分区,是 Linux 采取的一种内存策略,通过把磁盘的内存当做物理内存来用.这样做的目的就是增大了 物理内存的大小,其明显的缺点就是速度没有物理内存快. swap 区域一般分作两部分,一部分位于物理内存,叫做 swapcache 区域,另外一部分位于磁盘上. 这里涉及的到换页,简单来说就是把不经常使用的匿名页帧置换到磁盘上.注意,内存存在两种映射: 匿名映射和文件映射,文件映射还和虚拟文件系统有关,它有自己的文件缓存和缓冲,文件映射的物理页帧是不会来自 swap区域 所以 swap 区域就是匿名映射的物理页帧.这个区域由内核创建并管理.

碎片

  • 内部碎片是在一个已分配块比有效载荷大时发生的，例如块对齐。
  • 外部碎片是当空闲内存合计起来足够满足一个分配请求，但没有一个单独的空闲块足够大可以来处理这个请求时发生的。

隐式空闲列表

  • 堆由已分配块和空闲块组成。

  • 每个块的头部记录了块大小和是否分配。若加入双字的对其条件（这里1字是4字节），那么块大小就是8的倍数，即块头部最低3位为0。那么，我们就可以在块头部的最低1位记录该块是否已分配。假如块头部为(头部为1字，32位)

    0000 0000 0000 0000 0000 0000 0001 1001(2)=0x19=0x18|1,

    表明该块大小为18字节，且已分配。

空闲块操作

  • 分割：分配器找到一个匹配的空闲块时，需要选择分配的空间。一个选择是选择整个块，但会产生较大的内部碎片。另一个选择是将这个空闲块分割为两部分。
  • 合并：为解决假碎片问题，任何实际的分配器都必须合并相邻的空闲块。快速的分配器通常会选择某种形式的推迟合并。

Everything is a file. — Unix

  • 打开文件：应用程序通过要求内核打开相应的文件，来宣告它想要访问一个设备。内核返回描述符，它在后续对此文件的所有操作中标识这个文件。
  • 改变当前的文件位置：对每个打开的文件，内核保持一个文件位置 k，初始为0。这个文件位置是从文件开头起始的字节偏移量。
  • 读写文件：一个读操作就是从文件复制 n 个字节到内存，从当前位置 k 开始，然后将 k 增加到 k+n。给定一个大小为 m 字节0的文件，当 k≥m 时执行读操作会触发 EOF 条件，应用程序能检测到这个条件。类似地，写操作就是从内存复制 n 个字节到文件，从 k 开始，然后更新 k。
  • 关闭文件：当应用完成了对文件的访问后，它就通知内核关闭这个文件。对该文件所构建的数据结构等内存资源会被释放。

网络编程

C/S模型

  • 一个应用是由一个服务器和一个或者多个客户端进程组成。
  • 服务器管理某种资源，并且通过操作这种资源来为它的客户端提供服务。

  • 一个客户端-服务器事务（该事务非数据库事务，没有数据库事务的任何特性）由以下四步组成：

    1. 客户端发送请求给服务器，发起一个事务。
    2. 服务器收到请求后，解释之并操作其资源。
    3. 服务器给客户端发送响应，并等待下一个请求。
    4. 客户端收到响应并处理它。

网络

  • 客户端和服务器通常运行在不同的主机上，需要通过计算机网络的硬件和软件资源来通信。
  • 对主机而言，网络可以理解为一种I/O设备。
  • 网络信息通过插在I/O总线扩展槽的网络适配器（网卡）传输。

局域网（Local Area Network，LAN）

  • 以太网是流行的局域网技术。
  • 通过集线器连接的不同主机有同一网段。
  • 在同一局域网内的主机可以发送一段位（称为帧）到其他任何主机。每个主机适配器都能看到这个帧，但只有目的主机实际读取它。
  • 不同集线器可通过网桥连接，组成较大的局域网，称为桥接以太网。

广域网（Wide-Area Network，WAN）

  • 多个不兼容的局域网可以通过叫做路由器（router）的特殊计算机连接起来，组成一个internet（互联网络）。

 

  • 互联网络至关重要的性质是采用完全不同和不兼容的各种局域网和广域网组成。

  • 协议软件消除了不同网络之间的差异。这种协议必须提供两种基本能力：

    • 命名机制：不同的局域网技术有不同和不兼容的方式来为主机分配地址。互联网络协议通过定义一种一致的主机地址格式消除了这些差异。这个地址唯一地标识了这台主机。
    • 传送机制：在电缆上编码位和将这些位封装成帧方面，不同的联网技术有不同的和不兼容的方式。互联网络协议通过定义一种把数据位捆扎成不连续的片（称为包）的统一方式，从而消除了这些差异。

全球IP因特网

  • 全球IP因特网是最著名和最成功的互联网络实现。

  • 每台因特网主机都运行实现TCP/IP协议的软件，几乎每个现代计算机系统都支持这个协议。

  • IP协议提供基本的命名方法和传递机制。IP机制不可靠。

  • TCP是一个构建在TP之上 的复杂协议，提供了进程间可靠的全双工连接。

  • 我们可以将因特网看作一个世界范围的主机集合。

    • 主机被映射为一个32位IP地址。
    • IP地址被映射为因特网域名标识符。
    • 因特网主机上的进程能够通过连接和任何其他因特网主机上的进程通信。

IP地址

  • IP地址是一个32位无符号整数

因特网域名

  • 因特网定义了一组域名，以及一种将域名映射到IP地址的机制。
  • 映射是通过分布世界范围内的数据库（DNS）来维护的。

因特网连接

  • 一个连接是由它两端的套接字地址唯一确定的，这对套接字地址叫做套接字对。

套接字接口

  • socket：创建一个套接字描述符。return clientfd

  • connect：建立和服务器的连接。connect函数会阻塞，直到连接成功建立或是发生错误。得到sockaddr

  • bind listen accept：服务器用它们来和客户端建立连接。

    • bind：告诉内核将addr中的服务器套接字地址和套接字描述符

    • listen：服务器调用listen函数告诉内核，描述符是被服务器而不是客户端使用的。listen函数将sockfd从一个主动套接字转化为一个监听套接字

    • accept：等待来自客户端的连接请求，返回一个连接描述符connfd

      1. 客户端（clientfd） 服务器（listenfd(3)）：服务器阻塞在accept，等待监听描述符listenfd的连接请求
      2. 客户端（clientfd）--连接请求--> 服务器（listenfd(3)）：客户端通过调用和阻塞在connect，创建连接请求
      3. 客户端（clientfd）<--connfd--> 服务器（listenfd(3)）：服务器从accept返回connfd。客户端从connect返回。现在在clientfd和connfd之间已经建立起了连接
